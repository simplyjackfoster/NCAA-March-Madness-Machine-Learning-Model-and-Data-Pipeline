# Upset Prediction Improvement Design

## Goal

Improve tournament upset prediction from 8 upsets (current) to a realistic 11–18 range that matches historical averages, while identifying *which specific matchups* are likely upsets — not just getting the count right.

## Context

The pipeline currently predicts only 8 upsets per tournament vs. a historical average of 17.6 (range 11–23, 1985–2025). Root cause: the model is trained on synthetic game outcomes generated by `diff + rng.normal(0, 2) > 0`, which preserves the seed/rating hierarchy too faithfully and doesn't replicate real tournament variance or upset rates.

Real historical tournament data is already ingested (`MNCAATourneyCompactResults.csv`, `MNCAATourneySeeds.csv`, `MTeams.csv`) but unused for training.

## Architecture

Three coordinated changes:

```
Real Outcomes  →  Richer Features  →  Seed-Matchup Calibration  →  Prob Matrix
     ↑                  ↑                        ↑
MNCAATourneyResults  KenPom/Barttorvik    Historical upset rates
                     (already ingested)    by seed pair (1v16, etc.)
```

**Change 1 — Training data:** Replace synthetic labels with real tournament game results (~1,000 games, 2003–2024).

**Change 2 — Feature set:** Add three features that predict upsets independent of ratings: tempo mismatch, luck differential, strength-of-schedule differential.

**Change 3 — Calibration:** After Platt scaling, blend probabilities toward empirical per-seed-matchup win rates from historical data.

## Component Details

### `src/features/game_features.py` — Real Outcomes

Currently builds training rows with `label = int(diff + rng.normal(0, 2) > 0)`. Replace with:

1. Load `MNCAATourneyCompactResults.csv` (columns: `Season`, `WTeamID`, `LTeamID`)
2. Load `MNCAATourneySeeds.csv` to get seed for each team/year
3. Load `MTeams.csv` for `TeamID → TeamName` mapping
4. For each historical game, look up team features for that season and build a feature row with `label = 1` (winner won)
5. Add mirror row (`label = 0`) so the model sees both sides of each matchup

Only seasons where team features exist (2003–2024) are included. Games where team features cannot be joined are dropped with a warning.

### New Features in Feature Vector

Three new columns added to `game_features.py` matchup rows:

| Feature | Source column | Computation | Why it predicts upsets |
|---|---|---|---|
| `tempo_mismatch` | Barttorvik `adj_tempo` | `abs(tempo_a - tempo_b)` | Fast-paced underdogs create chaos vs slow favorites |
| `luck_diff` | KenPom `luck` | `luck_a - luck_b` | High-luck teams regress in tournament; model currently ignores this |
| `sos_diff` | Barttorvik SOS column | `sos_a - sos_b` | Weak-schedule favorites underperform vs battle-tested underdogs |

These columns are already present in ingested CSVs. `team_features.py` must expose them so `game_features.py` can compute differentials.

### `src/models/calibrate.py` — Seed-Matchup Calibration Layer

After Platt scaling, apply a blending step using empirical per-seed-matchup win rates:

```
final_prob = 0.7 × platt_prob + 0.3 × historical_seed_win_rate
```

Historical seed win rates are computed from `MNCAATourneyCompactResults.csv` + `MNCAATourneySeeds.csv` at calibration time, grouped by `(seed_a, seed_b)` pair. The 0.7/0.3 blend keeps the model's feature-driven signal dominant while preventing extreme probabilities (e.g. 1-seeds won't get 99.5%; 12-seeds won't get 2% when history says they win 35% of the time).

### `src/data/build_calibration_set.py`

Already reads Kaggle historical data. Update to use real outcomes consistent with the new `game_features.py` approach so calibration training data and prediction-time features match.

## Testing Approach

### Unit Tests

- `game_features.py`: assert no synthetic `rng.normal` calls in label generation; assert label column is binary (0/1 only); assert row count = historical game count × 2 (mirror rows)
- New features: assert `tempo_mismatch` ≥ 0 for all rows; assert `luck_diff` and `sos_diff` are finite (no NaN from KenPom join misses)
- Seed-matchup calibration: for 1v16 matchups, assert blended prob ∈ [0.90, 0.99]; for 8v9 matchups, assert blended prob ∈ [0.45, 0.55]

### Integration Test

Run full pipeline on 2025 data:
- Assert predicted upsets ≥ 11 (historical minimum)
- Assert predicted upsets ≤ 23 (historical maximum)
- Assert no single matchup probability exceeds 0.98 after calibration

### Regression Test

- `full_bracket_2026.txt` generates without error
- Champion name present in output
- Top-3 picks section renders correctly
